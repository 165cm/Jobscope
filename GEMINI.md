---

## 📝 コミットメッセージのルール

### 形式
```
絵文字 変更内容の要約（50文字以内）
```

### 使う絵文字
| 絵文字 | 意味              | 使う場面                       |
| ------ | ----------------- | ------------------------------ |
| ✨      | 新機能            | 新しい機能を追加したとき       |
| 🐛      | バグ修正          | バグを直したとき               |
| 🎨      | デザイン/見た目   | CSS変更、レイアウト調整        |
| ♻️      | リファクタリング  | コード整理（動作は変わらない） |
| 📝      | ドキュメント      | READMEなどの文書更新           |
| 🔧      | 設定              | 設定ファイルの変更             |
| 🗑️      | 削除              | 不要なファイル/コードを削除    |
| 🚀      | デプロイ/リリース | 本番公開時                     |

### 良い例
```
✨ ログイン機能を追加
🐛 ボタンが押せないバグを修正
🎨 ヘッダーの色を変更
```

### 悪い例
```
修正した        ← 何を修正したかわからない
update          ← 英語だし内容不明
いろいろ変更    ← 具体性がない
```

---

## ✅ コミット前のチェックリスト

Claude Codeがコミットする前に確認すること：

1. **動作確認**: エラーが出ないか
2. **console.log**: デバッグ用のログは削除したか
3. **コメント**: 必要なコメントは書いたか
4. **不要ファイル**: 余計なファイルが含まれていないか

---

## 🚫 コミットしないファイル

以下はコミットに含めないこと：

- `node_modules/` （npmパッケージ）
- `.env` （秘密の設定）
- `*.log` （ログファイル）
- `.DS_Store` （Mac のシステムファイル）

---

## 💬 コミュニケーション

- 日本語で会話する
- 専門用語は避けて、わかりやすく説明する
- コードを変更したら、何を変えたか簡単に説明する

- エラーが出たら、エラーメッセージ全文を見せて相談
- 「何がわからないかわからない」も OK、一緒に整理しましょう

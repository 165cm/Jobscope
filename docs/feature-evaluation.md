## 🎯 追加機能の優先順位（実装しやすい順）

### 🥇 優先度S：即実装可能・高付加価値

#### 1. 企業公式HP・採用ページの自動収集 ★★★★★

**実装難易度:** 🟢 低 (3-5時間)
**付加価値:** 🟡 中高 (8/10)
**技術的実装:**
- 会社名からGoogle検索APIまたはスクレイピングで企業HP URLを取得
- 採用ページ・会社概要ページのテキストを抽出
- AI解析で「企業文化」「福利厚生」「ミッション」を要約

**Notionフィールド追加案:**
```
- company_website: URL型
- company_description: Rich Text型（企業概要）
- benefits: Multi-select型（福利厚生タグ）
```

**実装コスト:**
- APIコスト: Google Custom Search API（無料枠: 100クエリ/日）
- 追加AI解析: ほぼゼロ（既存のプロンプトに追記）

**ユーザーメリット:**
- 企業の公式情報を一元管理
- 企業理念や文化を把握しやすい
- 面接準備に直結

#### 2. SNS検索リンク生成機能 ★★★★☆

**実装難易度:** 🟢 極低 (1-2時間)
**付加価値:** 🟡 中 (7/10)
**技術的実装:**
- 会社名から以下の検索URLを自動生成してNotionに埋め込み
  ```
  - X(Twitter): https://x.com/search?q={会社名}+評判
  - note: https://note.com/search?q={会社名}
  - LinkedIn: https://www.linkedin.com/search/results/companies/?keywords={会社名}
  ```

**Notionフィールド追加案:**
```
- search_x: URL型
- search_note: URL型
- search_linkedin: URL型
```

**実装コスト:** ほぼゼロ（URLテンプレート文字列のみ）

**ユーザーメリット:**
- ワンクリックでSNS調査開始
- 手作業の検索時間を大幅削減
- 定性的リサーチの導線確保

#### 3. OpenWork/Lighthouse/キャリコネへのディープリンク ★★★★☆

**実装難易度:** 🟢 極低 (1-2時間)
**付加価値:** 🟢 高 (9/10)
**技術的実装:**
- 会社名から口コミサイトの検索URLを自動生成


**Notionフィールド追加案:**
```
- openwork_url: URL型
- lighthouse_url: URL型
- careerconnection_url: URL型
```

**実装コスト:** ほぼゼロ

**ユーザーメリット:**
- 口コミサイトへ即アクセス
- 現社員・元社員の生の声を簡単確認
- **定性的リサーチで最も価値の高い情報源へのショートカット**

### 🥈 優先度A：実装可能・高付加価値

#### 4. gBizINFO API連携（企業基本情報） ★★★★☆

**実装難易度:** 🟡 中 (8-12時間)
**付加価値:** 🟢 高 (8/10)
**技術的実装:**
- [gBizINFO REST API](https://info.gbiz.go.jp/api/) を利用

  - 従業員数

  - 事業内容
  - 代表者名

**Notionフィールド追加案:**
```
- capital: Number型（資本金：百万円）
- founded_year: Number型（設立年）
- ceo_name: Text型（代表者名）
- business_description: Rich Text型（事業内容）
```

**実装コスト:** 無料（APIキー不要）

**ユーザーメリット:**
- 公的データベースから正確な企業情報を取得
- 企業規模・安定性の客観的判断
- スタートアップか大手かを即座に判別

### 🥉 優先度B：高難易度・高付加価値（中長期実装）

#### 6. 有価証券報告書（EDINET）データ取得 ★★★☆☆

**実装難易度:** 🔴 高 (20-30時間)
**付加価値:** 🟢 最高 (10/10)
**技術的実装:**
- [EDINET API](https://disclosure.edinet-fsa.go.jp/) を利用
- 会社名から企業コードを検索
- 最新の有価証券報告書から以下を抽出:
  - 平均年収（従業員の状況）
  - 平均勤続年数
  - 平均年齢
  - 離職率（計算可能な場合）
  - 自己資本比率
  - 営業利益率

**Notionフィールド追加案:**
```
- avg_salary_official: Number型（万円）
- avg_tenure: Number型（年）
- avg_age_official: Number型（歳）
- equity_ratio: Number型（%）
- operating_margin: Number型（%）
- edinet_url: URL型
```

**実装コスト:** 無料（APIキー不要）

**ユーザーメリット:**
- 最も信頼できる公式データ
- 年収・勤続年数の正確な数字
- 企業の財務健全性を客観評価
- **口コミサイトの不満と照合可能**

**技術的課題:**
- XBRL形式のパース（XML解析）
- 会社名→EDINETコードのマッピング
- 非上場企業は対象外

#### 7. OpenWork等の口コミスコア自動取得 ★★☆☆☆

**実装難易度:** 🔴 非常に高 (40-60時間 + リスク)
**付加価値:** 🟢 最高 (10/10)
**技術的実装:**
- OpenWork/Lighthouseのスクレイピング
- 以下のデータを抽出:
  - 総合評価スコア
  - 待遇面の満足度
  - 社員の士気
  - 風通しの良さ
  - 20代成長環境
  - 残業時間（実態）

**Notionフィールド追加案:**
```
- openwork_score: Number型（総合評価）
- openwork_growth: Number型（成長環境）
- openwork_worklife: Number型（ワークライフバランス）
- review_count: Number型（口コミ件数）
```

**実装コスト:** 高リスク

**技術的課題:**
- **利用規約違反リスク（スクレイピング禁止の可能性）**
- CAPTCHA/ログイン認証
- DOM構造の頻繁な変更
- IPブロックのリスク

**代替案:**
- ディープリンクのみ提供（優先度3）
- ユーザーが手動でスコアを入力
- 公式API提供待ち（現状なし）

**ユーザーメリット:**
- 最も価値の高い定性的データ
- 入社後のミスマッチ防止に直結
- ただし**実装リスクが高すぎるため非推奨**

---

#### 8. 就職四季報データ統合 ★☆☆☆☆

**実装難易度:** 🔴 非常に高 (著作権問題)
**付加価値:** 🟢 高 (8/10)
**技術的実装:**
- 就職四季報は書籍データ（デジタル版あり）
- 公式APIなし
- スクレイピング不可（著作権侵害）

**実装コスト:** 実質不可能

**技術的課題:**
- **著作権法違反リスク**
- データ入手方法がない
- 東洋経済新報社との契約が必要

**代替案:**
- ユーザーが手動で四季報データを入力
- Notionテンプレートに四季報用フィールドを用意

**ユーザーメリット:**
- 離職率・有給消化率などの貴重なデータ
- ただし**実装不可**
